## 4. Municipal Architecture (Era V)

### 4.1 City Hall (The Comptroller)
**Role:** Central Command & Governance.
The Comptroller is the brain of the city. It issues permits, manages the Treasury, and audits system processes.
*   **Key Feature:** **Dynamic Port Discovery**. City Hall assigns and publishes port allocations to prevent service collisions.
*   **Wallet Factory:** Features **Instant Generation** (BIP-39/SHA-256) for secure citizen onboarding.

### 4.2 Jimenez Military Base (Defense)
**Role:** Network Security & Threat Neutralization.
The Military Base monitors the city's perimeter (Network Ports).
*   **DEFCON System:** Enforces security protocols based on Governance Directives (currently **DEFCON 4**).
*   **Full Spectrum Scanning:** A new deep-scan protocol that monitors non-standard ports for "stealth" anomalies.
*   **Active Defense:** Automatically bans IPs engaging in brute-force attacks against critical gateways.

### 4.3 Jimenez Int'l Airport (Logistics)
**Role:** Transaction Processing & Packet Routing.
The Airport visualizes the Mempool as "Incoming Flights."
*   **Mempool as Airspace:** Every pending transaction is a plane seeking permission to land (be mined).
*   **Customs & Border Control:** Validates transaction signatures and gas limits before allowing entry.
*   **Postal Operations:** Manages data packets exchanged between service scripts.
*   **Blackbox Security:** Sensitive data is routed through secure "Target Block Bundle Flights," protected by Blackbox encryption.

## 6. Security & Privacy
*   **Dual-Layer USB Vaults:** Physical access keys required for critical operations.
*   **Local-Only Signing:** Private keys never leave the secure memory of the signing script.
*   **Stealth Mode:** USB deployment scripts use system attributes to mask critical files from the host OS.
*   **Sovereign Sync:** The offline Vault maintains a "One-Way Uplink" capability. When connected to a network-capable machine, it reads the `governance-directives.json` state from the Municipal Grid (City Hall) to update its DEFCON status and Security Protocols without ever exposing private keys to the internet.
*   **Protocol Omega:** A terminal security protocol for high-risk assets. USB Vaults can be flashed with a "Self-Destruct" directive. If specific conditions are met (Expiration Date reached or Fund Depletion detected), the Vault initiates a **Sector Overwrite Sequence**, corrupting all file headers and erasing the Master File Table (MFT), rendering the device physically irrecoverable.
